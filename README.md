# 🍻 数字淘汰 · 卡通版

一个**手机扫码即玩**的多人在线酒桌博弈小游戏，严格按照《数字淘汰 v5（最多20人版）》规则实现。
卡通画风、首次自动出新手教学、3–20 人同乐。

---

## 🚀 怎么开始（最简单方式）

> 只要一个人（房主）在电脑上启动，其他人用手机扫码就能玩。所有人连**同一个 WiFi**。

1. **双击 `启动游戏.bat`**
   - 第一次会自动安装依赖（需要电脑装了 [Node.js](https://nodejs.org)，下载 LTS 版一路下一步即可）。
   - 如果 Windows 弹出「防火墙」提示，请点**允许访问**（否则手机连不上）。
2. 窗口里会显示一行网址，例如：
   ```
   同一WiFi下手机访问： http://192.168.x.x:3000
   ```
   还会打印一个**二维码**，手机直接扫这个码也能打开。
3. **房主**：手机/电脑打开该网址 → 起名字 → 点「✨ 创建房间」，得到 4 位房间号和二维码。
4. **其他人**：手机打开同一网址 → 起名字 →
   - 扫房主屏幕上的二维码，或
   - 输入 4 位房间号 → 点「加入房间」。
5. 人到齐（≥3 人）后，房主点「🎮 开始游戏」。第一次玩建议**打开新手教学模式**，每一步都会有规则提示。

> 想结束：关闭那个黑色命令行窗口即可。

## 🎮 怎么玩（30秒速记）

- 每轮大家**同时**报一个数（1 到当前人数）。
- **撞车**（两人及以上报同一个数）→ 作废，这些人喝酒但留场。
- **放生**：没撞车的数字里，最小的若干名**通关**离场（人越多放越多）。
- **安然令牌🛡️**：留场玩家里报「最大独有数字」的人得 1 枚，喝酒前亮出可抵 1 杯（3 口）。
- 罚酒随小轮递增：第1-2轮1口、第3-4轮2口、第5轮起3口封顶。
- 剩 2 人 → **终局斗鸡**（报1或2）；双双报2 → **复活**邀请一名已通关者；接受则**三人最终局**（报1-3，最小独有数字逃脱）。

口诀：**小数抢逃生，大数换免死。** 详细规则点游戏里「📖 看规则 / 新手教学」。

## 📱 多设备说明

- **同一 WiFi**：最省事，扫码即玩，无需公网，无需登录。
- 刷新网页/手机锁屏不会掉队——会自动重连回原来的身份。
- 房主掉线时，房主身份会自动转给其他在线玩家，游戏不卡住。
- 想换台电脑当服务器也行，谁运行 `启动游戏.bat` 谁就是服务器。

## 🛠️ 给开发者

```bash
npm install      # 安装依赖
npm start        # 启动（默认端口 3000，可用 PORT 环境变量改）
npm test         # 运行游戏逻辑单元测试
```

文件结构：

| 文件 | 作用 |
|------|------|
| `game.js` | 纯游戏逻辑 / 状态机（不依赖网络，可单测） |
| `server.js` | 房间管理 + Socket.IO 实时同步 + 二维码 |
| `public/index.html` `style.css` `app.js` | 卡通风格前端（移动端优先，原生 JS） |
| `test/game.test.js` | 13 项规则单元测试 |

技术栈：Node + Express + Socket.IO（实时）+ qrcode（扫码加入）。前端零构建、零外网依赖，
Socket.IO 客户端由服务器本地提供，断网/纯局域网也能跑。

## ❓ 常见问题

- **手机打不开网址 / 转圈**：确认手机和电脑同一个 WiFi；检查电脑防火墙是否放行了 Node.js；
  有些「访客 WiFi」开了 AP 隔离会互相不通，换主 WiFi 即可。
- **端口被占用**：关掉占用 3000 端口的程序，或设 `PORT=4000` 再启动。
- **想重新看教学**：游戏里点「📖 看规则」，或清掉浏览器缓存后首次进入会自动播放。
